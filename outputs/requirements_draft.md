# Requirements Draft

## Business Requirements
- BR-1: Zvysit efektivitu prace uzivatelu sjednocenim roztistenych cinnosti.
- BR-2: Podporit strategii firmy pro odpadove spolecnosti resenim pokryvajicim workflow zakaznika.
- BR-3: Podporit rychle doruceni MVP pri omezene kapacite vyvoje.
- BR-4: Udrzet kompatibilitu se stavajicimi produkty a integracemi.

## Functional Requirements
- FR-1: System musi sjednotit vice kroku do jednoho uzivatelskeho rozhrani.
- FR-2: System musi automatizovat vybrane rutinni cinnosti.
- FR-3: System musi poskytovat dashboard stavu.
- FR-4: System musi umoznit integraci na existujici systemy pres dostupna rozhrani.
- FR-5: System musi zpistupnit klicova data na jednom miste.

## Non Functional Requirements
- NFR-1: Pouzitelnost - jednoduche pouziti pro hlavni uzivatelske role.
- NFR-2: Rozsiritelnost - navrh musi umoznit dalsi rozvoj.
- NFR-3: Cas-to-market - navrh musi podporit rychle MVP.
- NFR-4: Integracni kompatibilita - respektovat stavajici systemy a REST API.

## Otevrene otazky
- Kdo je primarni uzivatel MVP a jaky je jeho hlavni use-case?
- Ktera data jsou "klicova" pro centralni pohled?
- Ktere 2-3 integrace jsou povinne pro prvni fazi?
- Jake KPI pro MVP navazuji na firemni KPI (5 novych zakazniku mesicne)?
- Jake jsou provozni a bezpecnostni pozadavky (SLA, audit, role/prava)?
